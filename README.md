# Project Handoff Checklist

A standalone, zero-install webpage for running any software project through an MVP-first, phased checklist — from kickoff through post-launch. Open `index.html` in any browser. No build step, no server, no account. All data stays in your browser (`localStorage`); nothing is sent anywhere.

## What it does

- **Phases 0–5** — quick validation (with an explicit pass/park gate), idea explanation, design & planning, iterative implementation, polish & release, post-launch
- **MoSCoW feature board** — Must-have completion feeds Phase 3 progress
- **Two rigor modes** — Lightweight (personal projects) vs. Full (shared/professional), with per-item N/A + reasons, notes, and a "next up" pointer
- **Handoff Snapshot** — current state, next action, known issues, decision log
- **Portable state** — one-click versioned JSON export, JSON import, and handoff-markdown generation
- **Multi-project** — autosaved locally, light/dark theme

## Usage

Download or clone, then open `index.html` in a modern browser. Works offline. A pre-filled example project is available from the empty state.

## Philosophy

Built to make project intake and handoff survivable on low-capacity days: nothing depends on memory, every gate is explicit, and the current state of any project is always exportable in a form another person (or a future session) can pick up cold.
